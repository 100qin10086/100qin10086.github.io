Spring-JdbcTemplate

发表于2023-03-30|更新于2025-02-11|主流框架
|阅读量:

JdbcTemplate

1.需求 如果希望用spring框架来做项目,spring框架如何处理对数据库的操作

方案一 使用JDBC加德鲁伊数据库连接池

方案二 使用spring提供的操作数据库功能强大的类JdbcTemplate。我们可以同ioc容器来配置一个jdbcTemplate对象,使用它来完成对数据库表的各种操作

2.官方文档

JdbcTemplate APIs : Markdown语法。

2.JdbcTemplate-基本介绍

1.通过 Spring 可以配置数据源【德鲁伊,c3p0】，从而完成对数据表的操作

2.JdbcTemplate 是 Spring 提供的访问数据库的技术。可以将 JDBC 的常用操作封装为模板方法。【dbcTemplate 类图】

3.JdbcTemplate代码演示

使用步骤

1.创建配置文件 src/jdbc.properties

jdbc.userName=xxxx
jdbc.password=xxxx
jdbc.driverClass=com.mysql.jdbc.Driver
jdbc.url=jdbc:mysql://localhost:3306/spring

2.创建配置文件src/JdbcTemplate_ioc.xml

<!-- 引入外部属性文件 -->
<context:property-placeholder location="classpath:jdbc.properties"/>
<!-- 配置数据源 -->
<bean id="dataSource" class="com.mchange.v2.c3p0.ComboPooledDataSource">
<property name="user" value="${jdbc.userName}"></property>
<property name="password" value="${jdbc.password}"></property>
<property name="driverClass" value="${jdbc.driverClass}"></property>
<property name="jdbcUrl" value="${jdbc.url}"></property>
</bean>

使用JdbcTemplateTest向数据库中添加单行数据

@Test
public void addDataByJdbcTemplate(){
    ApplicationContext ioc = new ClassPathXmlApplicationContext("JdbcTemplate_ioc.xml");
    //获取jdbcTemplate对象
    JdbcTemplate jdbcTemplate = ioc.getBean(JdbcTemplate.class);
    //1.添加方式1 可能存在sql注入的问题
    String sqls="INSERT INTO monster VALUES (400,'红孩儿','枪法')";
    jdbcTemplate.execute(sqls);
    //2.添加方式二,防止sql注入
    String sql="INSERT INTO monster VALUES (?,?,?)";
    //affected,该sql语句执行后，数据库受影响的行数
    int affected = jdbcTemplate.update(sql, 500, "孙悟空", "七十二变");
    System.out.println(affected);
}

使用JdbcTemplateTest向数据库中数据进行修改

@Test
public void updateDataByJdbcTemplate(){
    ApplicationContext ioc = new ClassPathXmlApplicationContext("JdbcTemplate_ioc.xml");
    //获取jdbcTemplate对象
    JdbcTemplate jdbcTemplate = ioc.getBean(JdbcTemplate.class);
    //组织SQL
    String sql="UPDATE monster SET `kill`=? WHERE id=?";
    int affected = jdbcTemplate.update(sql, "美女计", "300");
    System.out.println(affected);
}

使用JdbcTemplate对象向数据库中批量添加数据

说明：再调API时，无法确定是哪一个方法，可以按照如下步骤进行选择【1】先确定名字【2】根据API提供相应的参数【3】把自己的调用思路清晰

以批量添加的方法为例，先确定方法名字，但是类中重写的方法太多了，此时我们再根据我们要实现的功能，以及能提供参数来确定方法

这是本次使用批量添加的方法的源码，我们可以发现它需要两个参数，一个是sql语句，而另外一个参数是List，此时我们就明白了，可以将要批处理的多条数据封装到list集合中

@Override
	public int[] batchUpdate(String sql, List<Object[]> batchArgs) throws DataAccessException {
		return batchUpdate(sql, batchArgs, new int[0]);
	}

使用JdbcTemplate对象向数据库中批量添加数据

@Test
public void addBatchDataByJdbcTemplate(){
    ApplicationContext ioc = new ClassPathXmlApplicationContext("JdbcTemplate_ioc.xml");
    //获取jdbcTemplate对象
    JdbcTemplate jdbcTemplate = ioc.getBean(JdbcTemplate.class);
    //1.先确定。猜测API名称
    String sql="INSERT INTO monster VALUES (?,?,?)";
    List<Object[]>batchArgs=new ArrayList<>();
    batchArgs.add(new Object[]{600,"老鼠精","偷吃粮食"});
    batchArgs.add(new Object[]{700,"老猫精","抓老鼠"});
    int[] ints = jdbcTemplate.batchUpdate(sql, batchArgs);
    for (int anInt : ints) {
        System.out.println(anInt);
    }
}

查询单行数据，并将其封装到对象中【再实际开发中非常重要】

代码分析

1.先确定方法名queryForObject
2.我们可以看到里面有个参数叫做RowMapper行处理器，和我们需要返回的一行数据将其封装到对象相对应
3.然后将我们需要封装的对象传入RowMapper方法

@Test
public void selectDataByJdbcTemplate(){
    ApplicationContext ioc = new ClassPathXmlApplicationContext("JdbcTemplate_ioc.xml");
    //获取jdbcTemplate对象
    JdbcTemplate jdbcTemplate = ioc.getBean(JdbcTemplate.class);
    //1.确定API的名字
    //猜测为public <T> T queryForObject(String sql, RowMapper<T> rowMapper) throws DataAccessException
    String sql="SELECT id AS monsterId,`name`,`kill` from monster WHERE id=100";
    //使用RowMapper 接口来对返回的数据，进行一个封装->底层使用的反射->setter
    //这里有一个细节；你查询的记录的表的字段需要和Monster的对象字段名保持一至
    RowMapper<Monster> monsterBeanPropertyRowMapper = new BeanPropertyRowMapper<Monster>(Monster.class);
    Monster monster = jdbcTemplate.queryForObject(sql, monsterBeanPropertyRowMapper);
    System.out.println(monster);
}

查询多行行数据，并将其封装到对象中【再实际开发中非常重要】

代码分析

1.看到是返回的是多行数据,所以先确定方法名,再看返回的对象 如：list,map,set之类的
2.再更具确定的方法名传入其需要的参数

@Test
public void selectMulDataByJdbcTemplate(){
    ApplicationContext ioc = new ClassPathXmlApplicationContext("JdbcTemplate_ioc.xml");
    //获取jdbcTemplate对象
    JdbcTemplate jdbcTemplate = ioc.getBean(JdbcTemplate.class);
    String sql="SELECT id AS monsterId,`name`,`kill` from monster WHERE id>?";
    RowMapper<Monster> monsterBeanPropertyRowMapper = new BeanPropertyRowMapper<Monster>(Monster.class);
    List<Monster> query = jdbcTemplate.query(sql, monsterBeanPropertyRowMapper,200);
    for (Monster monster : query) {
        System.out.println(monster);
    }
}

查询返回的结果只要一行一列

代码分析

1.因为是查询语句,所以先确定方法名
2.返回的是单行单列,所以方法返回值是Object
3.最后根据确定的方法传入需要的参数

@Test
public void selectScalarByJdbcTemplate(){
    ApplicationContext ioc = new ClassPathXmlApplicationContext("JdbcTemplate_ioc.xml");
    //获取jdbcTemplate对象
    JdbcTemplate jdbcTemplate = ioc.getBean(JdbcTemplate.class);
    String sql="SELECT `name` from monster WHERE id=?";
    //第二个参数表示返回单行单列的类型是什么
    String s = jdbcTemplate.queryForObject(sql, String.class, 100);
    System.out.println(s);
}

使用Map传入具名参数完成操作【我认为是对预处理的一种改进，将预处理中的?改为:字段名】

代码分析

1.其实就是将预处理中的?改为:参数名
2.并将需要到的参数封装到Map中
3.最后将Map传入方法

@Test
public void testDataByNameParameterJdbcTemplate(){
    ApplicationContext ioc = new ClassPathXmlApplicationContext("JdbcTemplate_ioc.xml");
    NamedParameterJdbcTemplate namedParameterJdbcTemplate = ioc.getBean(NamedParameterJdbcTemplate.class);
    //确定API
    //准备参数[:my_id, :name, :`kill`] 要求按照规定的名字来设置参数
    String sql="INSERT INTO monster VALUES(:id, :name, :kill)";
    Map<String,Object>paramMap=new HashMap<>();
    paramMap.put("id",800);
    paramMap.put("name","蚂蚁精");
    paramMap.put("kill","吃大米");
    int update = namedParameterJdbcTemplate.update(sql, paramMap);
    System.out.println(update);
}

使用sqlparametersoruce 来封装具名参数。

代码分析

1.将要添加的消息使用具名参数来封装
2.具名参数就是预处理的进阶版使用形式->:参数名

@Test
public void operDataBySqlparametersoruce(){
    ApplicationContext ioc = new ClassPathXmlApplicationContext("JdbcTemplate_ioc.xml");
    NamedParameterJdbcTemplate namedParameterJdbcTemplate = ioc.getBean(NamedParameterJdbcTemplate.class);
    //public int update(String sql, SqlParameterSource paramSource, KeyHolder generatedKeyHolder)
    String sql="INSERT INTO monster VALUES(:monsterId, :name, :kill)";
    Monster monster = new Monster(900, "狐狸精", "色诱之术");
    SqlParameterSource beanPropertySqlParameterSource = new BeanPropertySqlParameterSource(monster);
    int update = namedParameterJdbcTemplate.update(sql, beanPropertySqlParameterSource);
    System.out.println(update);
}
