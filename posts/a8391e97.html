实现任务阶段1-开发QgqDispatcher

说明：编写QgqDispatcherServlet充当原生的DispatcherServlet(即核心控制器);

分析示意图

代码实现

创建QgqDispatcherServlet

public class QgqDispatcherServlet extends HttpServlet {
    @Override
    protected void doGet(HttpServletRequest req, HttpServletResponse resp) throws
            ServletException, IOException {
        super.doGet(req, resp);
    }

    @Override
    protected void doPost(HttpServletRequest req, HttpServletResponse resp) throws
            ServletException, IOException {
        super.doPost(req, resp);
    }
}

创建qgqspringmvc.xml充当原来的applicationContext-mvc.xml文件(就是spring容器的配置文件，比如指定要扫描那些包下的类)

<?xml version="1.0" encoding="utf-8" ?>
<beans>
    <!--    指定要扫描的基本包以及子包的java类-->
    <component-scan base-package="com.qgq.controller,com.qgq.service"></component-scan>
</beans>

修改web.xml文件，完成对核心/前端控制器的配置

<!DOCTYPE web-app PUBLIC
        "-//Sun Microsystems, Inc.//DTD Web Application 2.3//EN"
        "http://java.sun.com/dtd/web-app_2_3.dtd" >

<web-app>
    <display-name>Archetype Created Web Application</display-name>
    <!--  配置QgqDispatcherServlet 作为我们自己的前端控制器-->
    <servlet>
        <servlet-name>QgqDispatcherServlet</servlet-name>
        <servlet-class>com.qgq.qgqspringmvc.servlet.QgqDispatcherServlet</servlet-class>
        <!--给QgqDispatcherServlet配置参数，指定要操作的spring容器的配置文件-->
        <init-param>
            <param-name>contextConfigLocation</param-name>
            <param-value>classpath:qgqspringmvc.xml</param-value>
        </init-param>
        <!--再tomcat启动时就自动加载-->
        <load-on-startup>1</load-on-startup>
    </servlet>
    <servlet-mapping>
        <!--因为QgqDispatcherServlet作为前端控制器,所以要拦截所有请求-->
        <servlet-name>QgqDispatcherServlet</servlet-name>
        <url-pattern>/</url-pattern>
    </servlet-mapping>
</web-app>

自定义RequestMapping注解

package com.qgq.qgqspringmvc.annotation;

import java.lang.annotation.*;

/**
 * @Author 秦家大胖
 * @Date 2023/2/11 15:21
 * @Version 1.0
 * RequestMapping 注解用于指定控制器-方法的映射路径
 */
@Target(ElementType.METHOD)//用于指定被修饰的Annotation能用于修饰那些元素
@Retention(RetentionPolicy.RUNTIME)//用于指定该Annotation可以保留多长时间，编译器把注解记录在class文件中，当运行Java程序时，JVM会保留注释通过反射获取该注解
@Documented//在生成文档时，可以看到该注解
public @interface RequestMapping {
    String value() default "";
}

编写XMLParser工具类，可以解析qgqspringmvc.xml,得到要扫描的包

思路分析

1.创建getbasePackage(String xmlFile)传入xml文件名
2.使用dom4j技术对xml文件经行读取
3.要注意”component-scan”是根目录下bean下的一个子目录，而”base-package”是”component-scan”目录下的一个属性

package com.qgq.qgqspringmvc.xml;

import org.dom4j.Document;
import org.dom4j.DocumentException;
import org.dom4j.Element;
import org.dom4j.io.SAXReader;

import java.io.InputStream;

/**
 * @Author 秦家大胖
 * @Date 2023/4/7 14:38
 * @Version 1.0
 * 我亦无他，惟手熟尔
 */
public class XMLParser {
    public static String getBasePackage(String xmlFile) {
        //整个解析的过程是前面讲过的
        SAXReader saxReader = new SAXReader();
        //通过得到类的加载路径->获取到spring配置文件
        ClassLoader classLoader = XMLParser.class.getClassLoader();
        InputStream resourceAsStream = classLoader.getResourceAsStream(xmlFile);
        try {
            //得到xml文件的文档
            Document document = saxReader.read(resourceAsStream);
            Element rootElement = document.getRootElement();
            Element componentScanElement = rootElement.element("component-scan");
            String basePackage = componentScanElement.attribute("base-package").getText();
            return basePackage;
        } catch (DocumentException e) {
            throw new RuntimeException(e);
        }
    }
}

开发QgqWebApplicationContext，充当Spring容器，得到扫描类的全路径列表，完成功能说明-把指定的目录包括子目录下的Java类的全路径扫描到集合中

思路分析

1.根据上面写的XMLParser工具类，得到要扫描的包
2.因为要扫描的包可能不知不止一个，所以再配置文件中将要扫描的包用,号隔开，所以将得到的要扫描的包同样用,号隔开，这样就将全部要扫描的包得到
3，调用方法scanPackage方法对各个包进行扫描,获取到classFullPathList
4.scanPackage分析先通过类加载器url=this.getClass().getClassLoader().getResource(“/“ + pack.replaceAll(“\.”, “/“))得到真正执行的类，形式为D:\idea_java_projects\hsp-springmvc\target\hsp-springmvc\WEB-INF\classes\com\hspedu\controller\MonsterController.class
5.取出路径(url.getFile),因为路径也可以作为文件,所以File dir = new File(path);
6.扫描这个目录，及其子目录dir.listFile(),如果是目录,则递归扫描(需要带上子目录的目录名),如果是文件，获取到类的全路径，去掉.class放入到集合中，因为这些路径到时候要通过反射实例化对象

public void scanPackage(String pack) {
        //通过类的加载器得到指定的包对应的工作路径[绝对路径]
        URL resource = this.getClass().getClassLoader().getResource("/" + pack.replace(".", "/"));
        //根据得到的路径对其进行扫描，吧类的全路径保存到classFullPathList
        String path = resource.getFile();
        //再io中把目录也视为一个文件
        File file = new File(path);
        //遍历file
        for (File f : file.listFiles()) {
            if (f.isDirectory()) {
                scanPackage(pack + "." + f.getName());
            } else {
                String classFullPath = pack + "." + f.getName().replace(".class", "");
                classFullPathList.add(classFullPath);
            }
        }
    }

完善QgqWebApplicationContext，充当Spring容器-实例化对象到容器中

完成功能说明，将扫描到的类，再满足条件的情况下(即有相应的注解@Controller@Service),反射注入到ioc容器

思路分析

1.创建一个类型为 ConcurrentHashMap的ioc用于存放反射后的bean对象
2.初始化自定义的spring容器，目标就是要把@Controller等等初始化到容器中
3.初始化具体过程，再init中先完成得到要扫描的包，以及将要扫描的包极其子包中文件放入到Arraylist中，再实例化扫描到的类
4.实例化扫描到的类，先遍历存放路径的集合，通过反射创建实例，再根据实例判断其是否有@Controller注解
5.如果类有@Controller注解，得到该类的类名(首字母小写)作为key，再得到实例化对象作为value放入到ioc中

public void executeInstance() {
       //判断是否扫描到了类
       if (classFullPathList.size() == 0) {
           return;
       }
       try {
           for (String classFullPath : classFullPathList) {
               Class<?> clazz = Class.forName(classFullPath);
               if (clazz.isAnnotationPresent(Controller.class)) {
                   //得到类名数字目小写
                   String simpleName = clazz.getSimpleName();
                   simpleName = StringUtils.uncapitalize(simpleName);
                   ioc.put(simpleName, clazz.newInstance());
               }
            }
       } catch (Exception e) {
           e.printStackTrace();
       }
   }
