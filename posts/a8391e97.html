编写XMLParser工具类，可以解析qgqspringmvc.xml,得到要扫描的包

思路分析

1.创建getbasePackage(String xmlFile)传入xml文件名
2.使用dom4j技术对xml文件经行读取
3.要注意”component-scan”是根目录下bean下的一个子目录，而”base-package”是”component-scan”目录下的一个属性

package com.qgq.qgqspringmvc.xml;

import org.dom4j.Document;
import org.dom4j.DocumentException;
import org.dom4j.Element;
import org.dom4j.io.SAXReader;

import java.io.InputStream;

/**
 * @Author 秦家大胖
 * @Date 2023/4/7 14:38
 * @Version 1.0
 * 我亦无他，惟手熟尔
 */
public class XMLParser {
    public static String getBasePackage(String xmlFile) {
        //整个解析的过程是前面讲过的
        SAXReader saxReader = new SAXReader();
        //通过得到类的加载路径->获取到spring配置文件
        ClassLoader classLoader = XMLParser.class.getClassLoader();
        InputStream resourceAsStream = classLoader.getResourceAsStream(xmlFile);
        try {
            //得到xml文件的文档
            Document document = saxReader.read(resourceAsStream);
            Element rootElement = document.getRootElement();
            Element componentScanElement = rootElement.element("component-scan");
            String basePackage = componentScanElement.attribute("base-package").getText();
            return basePackage;
        } catch (DocumentException e) {
            throw new RuntimeException(e);
        }
    }
}

开发QgqWebApplicationContext，充当Spring容器，得到扫描类的全路径列表，完成功能说明-把指定的目录包括子目录下的Java类的全路径扫描到集合中

思路分析

1.根据上面写的XMLParser工具类，得到要扫描的包
2.因为要扫描的包可能不知不止一个，所以再配置文件中将要扫描的包用,号隔开，所以将得到的要扫描的包同样用,号隔开，这样就将全部要扫描的包得到
3，调用方法scanPackage方法对各个包进行扫描,获取到classFullPathList
4.scanPackage分析先通过类加载器url=this.getClass().getClassLoader().getResource(“/“ + pack.replaceAll(“\.”, “/“))得到真正执行的类，形式为D:\idea_java_projects\hsp-springmvc\target\hsp-springmvc\WEB-INF\classes\com\hspedu\controller\MonsterController.class
5.取出路径(url.getFile),因为路径也可以作为文件,所以File dir = new File(path);
6.扫描这个目录，及其子目录dir.listFile(),如果是目录,则递归扫描(需要带上子目录的目录名),如果是文件，获取到类的全路径，去掉.class放入到集合中，因为这些路径到时候要通过反射实例化对象

public void scanPackage(String pack) {
        //通过类的加载器得到指定的包对应的工作路径[绝对路径]
        URL resource = this.getClass().getClassLoader().getResource("/" + pack.replace(".", "/"));
        //根据得到的路径对其进行扫描，吧类的全路径保存到classFullPathList
        String path = resource.getFile();
        //再io中把目录也视为一个文件
        File file = new File(path);
        //遍历file
        for (File f : file.listFiles()) {
            if (f.isDirectory()) {
                scanPackage(pack + "." + f.getName());
            } else {
                String classFullPath = pack + "." + f.getName().replace(".class", "");
                classFullPathList.add(classFullPath);
            }
        }
    }

完成请求URL和控制器方法的映射关系

完成功能说明 创建一个类，将@Controller修饰的bean，@RequestMapping的url路径，以及@RequestMapping修饰的方法封装到一个对象中

示意图分析

思路分析

1.创建一个QgqHandler类，里面存放的为@RequestMapping的url和对应的控制器-方法映射关系保存到集合中
2.在自己创建的核心控制器 QgqDispatcherServlet中创建ArrayList集合，存放url和控制器-方法的映射关系
3.创建方法initHandlerMapping，完成控制器层url—>Controller—>方法的映射关系(该关系封装到QgqHandler对象)，并放入到handlerList集合中
4.initHandlerMapping方法的具体的操作，先遍历自定义容器中的ioc判断里面的实例化对象是否有@Controller注解
5.如果有@Controller注解，得到该实例化的所有方法并遍历，判断这些方法是否有注解@RequestMapping。
6.如果有@RequestMapping注解，得到其的value值，并将url-控制器-方法封装到QgqHandler对象中，最后放入到ArrayList中

private void initHandlerMapping() {
        if (qgqWebApplicationContext.ioc.isEmpty()) {
            //判断当前ioc容器是否为空
            return;
        }
        //遍历ioc容器的bean对象，然后进行url映射器处理
        Set<Map.Entry<String, Object>> entries = qgqWebApplicationContext.ioc.entrySet();
        for (Map.Entry<String, Object> entry : entries) {
            Class<?> clazz = entry.getValue().getClass();
            //如果注入的Bean是Controller
            if (clazz.isAnnotationPresent(Controller.class)) {
                //取出所有的方法
                Method[] declaredMethods = clazz.getDeclaredMethods();
                //遍历
                for (Method declaredMethod : declaredMethods) {
                    if (declaredMethod.isAnnotationPresent(RequestMapping.class)) {
                        String url =declaredMethod.getAnnotation(RequestMapping.class).value();
                        QgqHandler qgqHandler = new QgqHandler(url, entry.getValue(), declaredMethod);
                        handlerList.add(qgqHandler);
                    }
                }
            }
        }
    }

private QgqHandler getQgqHandler(HttpServletRequest request){
        //先回去到用户请求的uri
        String requestURI = request.getRequestURI();
        //遍历
        for (QgqHandler qgqHandler : handlerList) {

实现任务阶段 6- 完成 Spring 容器对象的自动装配 -@Autowried

说明：完成Spring容器中对象的注入/自动装配

完成任务说明:加入@AutoWired注解，进行对象属性的装配

思路分析

1.先写一个注解@AutoWired
2.在初始化方法中将扫描到的类反射到ioc的容器中后
3.接下来就是完成注入的bean对象的属性的装配
4.遍历ioc中的所有注入的bean对象，得到其中的所有的字段
5.遍历得到的这些字段，判断是否有注解@AutoWired,
6.如果有@AutoWired注解，并且@AutoWired有value，在ioc容器中得到这个value的实例化对象，通过反射装配对象
7.如果没有value，得到字段类型的首字母小写作为名字在ioc容器中查找，如果找到，通过反射装配对象。
8.在装配时，要进行爆破，因为有的字段是私有类型

public void executeAutoWired() {
       //判断ioc有没有要装配的对象
       if (ioc.isEmpty()) {
           return;
       }
       //遍历ioc中的所有注入的bean对象，然后获取到bean的所有字段，判断是否需要装配
       /**
        * 一个entry=>(String,Object) String就是注入对象时名称，object就是bean
        */
       for (Map.Entry<String, Object> entry : ioc.entrySet()) {
           //String key = entry.getKey();
           Object bean = entry.getValue();
           Field[] declaredFields = bean.getClass().getDeclaredFields();
           for (Field declaredField : declaredFields) {
               if (declaredField.isAnnotationPresent(AutoWired.class)) {
                   AutoWired annotation = declaredField.getAnnotation(AutoWired.class);
                   String beanName = annotation.value();
                   if ("".equals(beanName)) {
                       //即得到字段类型的首字母小写作为名字
                       String simpleName = declaredField.getType().getSimpleName();
                       beanName = StringUtils.uncapitalize(simpleName);
                   }
                   //如果设置了value
                   if (null == ioc.get(beanName)) {
                       throw new RuntimeException("ioc容器不存在要装配的bean");
                   }
                   //为了防止属性时private，需要暴力破解
                   declaredField.setAccessible(true);
                   //可以装配属性
                   try {
                       declaredField.set(bean, ioc.get(beanName));
                   } catch (Exception e) {
                       throw new RuntimeException(e);
                   }
               }
           }
       }
   }

实现任务阶段 7- 完成简单视图解

功能说明通过方法返回的 String, 转发或者重定向到指定

思路分析

1.在QgqDispatcherServlet中的doPost方法中编写一个请求转发的方法，因为此时的servlet刚好被调用
2.executeDispatch(HttpServletRequest request, HttpServletResponse response)方法解析
3.先通过传入的request从存放控制器-url-方法映射集合中读取到一组映射【其实就是一个QgqHandler对象】
4.判断这组映射是否为空，如果为空，response返回404
5.如果不为空，先通过qgqHandler这个对象得到目标方法的所有形参信息
6.创建一个参数数组【对应实参数组】
7.遍历形参数组，按照形参的类型先将HttpServletRequest和HttpServletResponse按照形参的顺序，放入到实参数组中
8.得到request传入的所有的参数，是以Map类型接收的
9.再将这些参数按照形参数组的顺序进行填充到实参数组中，没有传入的则按照默认填充
10.反射调用目标方法，得到目标方法的返回结果 qgqHandler.getMethod().invoke
11.判断返回的结果是forward还是redirect，按照这个进行请求转发或者重定向

private void executeDispatch(HttpServletRequest request, HttpServletResponse response) {
        QgqHandler qgqHandler = getQgqHandler(request);
        try {
            if (null == qgqHandler) {//说明用户请求的资源不存在
                response.getWriter().print("<h1>404 NOT FOUND</h1>");
            } else {//匹配成功,反射调用控制器的方法
                //1.得到目标方法的形参信息[对应的数组]
                Class<?>[] parameterTypes = qgqHandler.getMethod().getParameterTypes();
                //2.创建一个参数数组[对应实参数组]，再后面反射调用目标方法时，会使用到
                Object[] params = new Object[parameterTypes.length];
                //遍历形参数组，根据形参数组信息，将实参填充到实参数组
                for (int i = 0; i < parameterTypes.length; i++) {
                    //取出每一个形参类型
                    Class<?> parameterType = parameterTypes[i];
                    //如果这个形参是HttpServletRequest将request填充到params
                    if ("HttpServletRequest".equals(parameterType.getSimpleName())) {
                        params[i] = request;
                    } else if ("HttpServletResponse".equals(parameterType.getSimpleName())) {
                        params[i] = response;
                    }
                }
                //将Http用户请求参数封装到params数组中，注意，再填充数组时，注意顺序问题
                //为什么可以反射，因为method=public void com.qgq.controller.MonsterController.listMonster(javax.servlet.http.HttpServletRequest,javax.servlet.http.HttpServletResponse)
                //有该方法的全路径
                //将需要传递给目标方法的实参=>封装到实参数组=>然后以反射的方式传入给目标方法
                //1.获取http请求参数集合
                //解析String, String[]
                //String表示http请求的参数名
                //String[]表示http请求的参数值
                //处理提交的的数据中文乱码
                request.setCharacterEncoding("utf-8");
                Map<String, String[]> parameterMap = request.getParameterMap();
                //遍历parameterMap将请求参数，按照顺序填充到实参数组params
                for (Map.Entry<String, String[]> stringEntry : parameterMap.entrySet()) {
                    String name = stringEntry.getKey();
                    String value = stringEntry.getValue()[0];
                    //我们得到请求的参数对应目标方法的第几个形参，将其填充
                    int indexRequestParameterIndex = getIndexRequestParameterIndex(qgqHandler.getMethod(), name);
                    if (indexRequestParameterIndex != -1) {//找到对应的位置
                        params[indexRequestParameterIndex] = value;
                    } else {//没有找到@RequestParam注解对应的参数，就会使用默认的机制进行配置
                        //思路
                        //1.得到目标方法的所有形参的名称
                        //2.对得到目标方法的形参名进行遍历,如果匹配就把当前请求的参数值，填充到params
                        List<String> parameterNames = getParameterNames(qgqHandler.getMethod());
                        for (int i = 0; i < parameterNames.size(); i++) {
                            //如果请求参数名和目标方法的形参名一样,说明匹配成功
                            if (name.equals(parameterNames.get(i))) {
                                params[i] = value;
                                break;
                            }
                        }
                    }
                }
                //反射调用目标方法，得到目标方法返回的结果
                Object result = qgqHandler.getMethod().invoke(qgqHandler.getController(), params);
                if (result instanceof String) {
                    String viewName = (String) result;
                    if (viewName.contains(":")) {
                        String viewType = viewName.split(":")[0];
                        String viewPage = viewName.split(":")[1];
                        //判断是forward还是redirect
                        if ("forward".equals(viewType)) {
                            request.getRequestDispatcher(viewPage).forward(request, response);
                        } else if ("redirect".equals(viewType)) {
                            response.sendRedirect(viewName);
                        }
                    } else {//默认是请求转发
                        request.getRequestDispatcher(viewName).forward(request, response);
                    }
                }//这里还可以扩展
            }
        } catch (Exception e) {
            throw new RuntimeException(e);
        }
    }
